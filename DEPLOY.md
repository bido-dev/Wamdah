# دليل تشغيل ونشر مشروع "ومضة" (Wamda) ⚡

تطبيق **ومضة** مبني باستخدام إطار العمل **Next.js (App Router)** وبدعم كامل للغة العربية ومواءمة بصرية رائعة مع هوية كلية علوم الحاسب والمعلومات بجامعة الملك سعود.

---

## 1. التشغيل المحلي (Development Mode)

لتشغيل التطبيق وتجربته على جهازك المحلي:

1. افتح مبدل الأوامر (Terminal) في مجلد المشروع.
2. قم بتثبيت الحزم البرمجية (إذا لم تكن مثبتة):
   ```bash
   npm install
   ```
3. ابدأ تشغيل خادم التطوير:
   ```bash
   npm run dev
   ```
4. افتح المتصفح على الرابط: [http://localhost:3000](http://localhost:3000)

> **💡 نصيحة للتجربة:** افتح نافذتين متصفح بجوار بعضهما البعض (واحدة متصفح خفي). في النافذة الأولى اختر "استقبال" وفي الثانية اختر "إرسال"، وأدخل الرمز لتجربة رفع الملفات والروابط واستقبالها فورياً.

---

## 2. الرفع إلى GitHub 🚀

لرفع الكود إلى حسابك الشخصي على منصة GitHub:

1. توجه إلى [GitHub](https://github.com) وأنشئ مستودعاً جديداً (New Repository) باسم `wamda`.
2. لا تقم بإضافة ملفات README أو `.gitignore` جديدة (المشروع يحتوي عليها بالفعل).
3. في مجلد المشروع المحلي، قم بتنفيذ الأوامر التالية (استبدل `YOUR_USERNAME` باسم حسابك):
   ```bash
   # إضافة خادم ريموت الخاص بـ GitHub
   git remote add origin https://github.com/YOUR_USERNAME/wamda.git
   
   # تعيين الفرع الرئيسي
   git branch -M main
   
   # رفع الكود
   git push -u origin main
   ```

---

## 3. النشر على Vercel ☁️

منصة Vercel هي البيئة الرسمية والمثالية لنشر تطبيقات Next.js بضغطة زر وبشكل مجاني:

1. سجل الدخول إلى موقع [Vercel](https://vercel.com) (يفضل باستخدام حساب GitHub الخاص بك).
2. اضغط على زر **"Add New..."** ثم اختر **"Project"**.
3. ستظهر لك قائمة بمستودعات GitHub الخاصة بك؛ ابحث عن مستودع `wamda` واضغط على **"Import"**.
4. اترك جميع الإعدادات الافتراضية كما هي (Next.js يتم التعرف عليه تلقائياً).
5. اضغط على زر **"Deploy"**.
6. خلال أقل من دقيقة، سيكتمل بناء التطبيق ويمنحك رابط ويب عام ومجاني للمشروع (مثل: `wamda.vercel.app`).

---

## 4. البنية السحابية وإمكانيات التطوير المستقبلي 🛠️

- **آلية العمل الحالية (Local Mock):**
  يقوم التطبيق بحفظ بيانات الجلسات في ملف JSON ورفع الملفات مؤقتاً داخل مجلد `/tmp` التابع للخادم (وهو مجلد مسموح بالقراءة والكتابة فيه على خوادم Vercel Serverless). يتم مسح البيانات فور التحميل أو بعد 5 دقائق.
  
- **الترقية إلى قاعدة بيانات سحابية (Production Grade):**
  إذا كنت ترغب في تشغيل التطبيق لآلاف الطلاب بشكل متزامن بكفاءة قصوى، يمكنك استبدال محركات `src/lib/db.ts` بـ:
  1. **Supabase / Firebase Realtime Database**: لحفظ الرموز واستقبال التغييرات بشكل لحظي بدون Polling.
  2. **Supabase Storage / Vercel Blob**: لحفظ الملفات المرفوعة مؤقتاً مع إعداد تذكرة حذف تلقائي (TTL) بعد 5 دقائق لضمان مسحها.
